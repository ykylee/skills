### 핵심 결론 (synthesis)

**antd 도입 보류 (do not adopt antd directly)** — migration 비용 *very high*.

| 평가 축 | 결과 | 비고 |
|---|---|---|
| Bundle size | **~10x** 우리 deck 대비 | 555 kB (gzip 178 kB) vs ~20-50 kB |
| Build step | **필수** | Vite/Webpack + React JSX transpile |
| Runtime | **React 18 + react-dom + antd** | vanilla port 가능성 ✗ |
| Theming | cssVar `:where(hash-class)` + runtime-hash-knowledge | static-authoring ✗ |
| Static-extract tool | `@ant-design/static-style-extract` (SSR 전용) | vanilla HTML 진입점 ✗ |
| 1920x1080 PDF | orthogonal (blocker 아님) | reveal.js 등은 자연 지원 |

### 권고

- **antd 직접 도입 보류** — React runtime + build pipeline + no-build 손실.
- **대안 1**: UnoCSS CDN runtime (진정한 no-build, `<script src="...@unocss/runtime">`,
  단 FOUC + runtime JS overhead caveat).
- **대안 2**: ASTRYX 정합 hand-authored minimal CSS layer — 이미 §A hybrid 정책으로
  구현됨 (4-layer cascade + tokens layer + Open Props absorption). 정합도 최상.
